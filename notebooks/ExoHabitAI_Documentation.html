Handling Missing Data

04

Two imputation strategies are used based on feature type. Rows that are completely missing across all key features are removed outright before imputation.

Fig 3: Imputation strategy applied per feature. Blue = Median, Orange = Mode.
Feature TypeFeaturesMethodRationale
Numeric ‚Äî Planetary pl_rade, pl_bmasse, pl_orbper, pl_orbsmax, pl_eqt, pl_dens Median Imputation Median is robust to skewed distributions common in planetary data (many small planets, few giant ones)
Numeric ‚Äî Stellar st_teff, st_lum, st_met Median Imputation Star temperatures are heavily skewed; median prevents hot outlier stars distorting imputed values
Categorical st_spectype Mode Imputation The most common star type (typically M-type) is used to fill missing spectral class values
Fully Missing Rows All features Row Removal Records missing all usable features are dropped as they provide no signal 
# Median imputation for numeric features
for col in ['pl_rade','pl_bmasse','pl_orbper','pl_orbsmax','pl_eqt','pl_dens','st_teff','st_lum','st_met']:
    df[col] = df[col].fillna(df[col].median())

# Mode imputation for categorical
df['st_spectype'] = df['st_spectype'].fillna(df['st_spectype'].mode()[0])

# Verify ‚Äî all zeros after imputation
df.isnull().sum()  # ‚Üí all 0

Outlier Detection & Removal

05

Two complementary methods are used. Physically impossible values are removed first (hard filters), then statistical outliers are capped or removed using IQR.

Method 1: Z-Score

Identifies values more than 3 standard deviations from the mean. Applied for approximately Gaussian features. Flags statistically improbable measurements.

Method 2: IQR (Interquartile Range)

Applied to pl_rade. Values below Q1 ‚àí 1.5√óIQR or above Q3 + 1.5√óIQR are removed. Robust to skewed distributions.

Fig 4: Planet radius distribution before (left) vs after (right) IQR-based outlier removal. Extreme values representing physically unrealistic giant planets are eliminated.

pl_orbper > 0Non-positive orbital period ‚Üí impossibleOrbital period must be positive (in days)
pl_orbsmax > 0Non-positive semi-major axis ‚Üí impossibleDistance from star must be positive (in AU)
pl_eqt > 0Equilibrium temp ‚â§ 0K ‚Üí impossibleAbsolute temperature must exceed 0K (‚àí273¬∞C)
IQR on pl_radeValues outside Q1‚àí1.5√óIQR ‚Ä¶ Q3+1.5√óIQRRemoves extreme gas giant radius outliers
 
# Step 1: Remove physically impossible values
df = df[df['pl_rade'] > 0]
df = df[df['pl_bmasse'] > 0]
df = df[df['pl_orbper'] > 0]
df = df[df['pl_orbsmax'] > 0]
df = df[df['pl_eqt'] > 0]

# Step 2: IQR on pl_rade
Q1 = df['pl_rade'].quantile(0.25)
Q3 = df['pl_rade'].quantile(0.75)
IQR = Q3 - Q1
df = df[(df['pl_rade'] >= Q1 - 1.5*IQR) & (df['pl_rade'] <= Q3 + 1.5*IQR)]

Feature Engineering

07

Three domain-informed composite features are created to encode physical habitability signals into ML-readable numeric form, based on known astrophysics of potentially habitable planets.

Fig 5: Summary of all engineered features, their formulas, and purpose in the ML pipeline.

1. Habitability Score Index

Computes the total "distance" from ideal Earth-like conditions across four dimensions: equilibrium temperature (target: 288K), planet radius (target: 1 R‚äï), orbital distance (target: 1 AU), and stellar luminosity. A lower score means the planet is more Earth-like.

df['habitability_score'] = (
    abs(df['pl_eqt'] - 288) +     # ‚Üê distance from 288K
    abs(df['pl_rade'] - 1) +      # ‚Üê distance from 1 R‚äï
    abs(df['pl_orbsmax'] - 1) +   # ‚Üê distance from 1 AU
    abs(df['st_lum'])             # ‚Üê luminosity deviation
)

2. Stellar Compatibility Index

Normalizes the host star's effective temperature against the maximum observed temperature in the dataset. Values near 1.0 indicate very hot stars; values near 0.0 indicate cool stars like M-dwarfs. This converts stellar temperature into a bounded 0‚Äì1 ratio for ML use.

df['stellar_compatibility'] = df['st_teff'] / df['st_teff'].max()

3. Orbital Stability Factor

Divides the semi-major axis by the orbital period, creating a ratio that captures orbital dynamics. Planets with consistent, stable orbits (low eccentricity implied by constant ratio) tend to have more predictable climates and better long-term habitability prospects.

df['orbital_stability'] = df['pl_orbsmax'] / df['pl_orbper']

Categorical Encoding & Feature Scaling

08

One-Hot Encoding ‚Äî st_spectype

The spectral type column (G, K, M, F, A, etc.) is expanded into binary dummy columns ‚Äî one column per unique star type. This prevents the model from treating star types as ordered (e.g. G > K > M).

Result: 1 categorical column ‚Üí 261 binary columns (one per unique spectral subtype found in data)

df = pd.get_dummies(df, columns=['st_spectype'])

StandardScaler ‚Äî All Numerics

All float and integer columns are scaled to zero mean and unit variance using scikit-learn's StandardScaler. This ensures features on different scales (e.g., orbital period in days vs. temperature in K) contribute equally to ML models.

Formula: z = (x ‚àí Œº) / œÉ for each feature

from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
numeric_cols = df.select_dtypes(include=['float64','int64']).columns
df[numeric_cols] = scaler.fit_transform(df[numeric_cols])

Target Variable Creation

09

A binary classification target is created from the engineered habitability_score. Planets with a habitability score below the median are labeled as potentially habitable (class 1); those above are labeled non-habitable (class 0). This produces a perfectly balanced dataset.

Fig 6: Target class distribution. Both classes contain exactly 13,801 records ‚Äî a perfect 50/50 balance, ideal for binary classification training.
# Binary target: 1 = potentially habitable (lower habitability_score = more Earth-like)
df['target'] = (df['habitability_score'] < df['habitability_score'].median()).astype(int)
